🔍 What this HUD does

RLV Scan is a Mentor welfare tool for identifying unwanted RLV control on a resident's avatar in Firestorm. Touch the HUD, select the resident from a nearby avatar menu, and within seconds you receive a full IM report.

The resident receives a notification in local chat before the scan begins.

Phase 1 Finds all nearby avatars sorted by distance and presents them as a touch menu. You pick who to scan.
Phase 2 Calls llGetAttachedList() on the selected avatar across three passes, assessing each attachment by name, description, owner, and script count against a database of known RLV products.
Phase 3 Pings the RLVrs relay channel -1812221819. If an active relay is present it forwards the query to the resident's Firestorm viewer, which responds with its RLVa version string. Catches relay HUDs invisible to the attachment scan.
🕹️ How to use it
1
Wear the HUD.
2
Stand near the resident you are helping.
3
Touch the HUD — a menu appears listing nearby avatars sorted by distance.
4
Tap the resident's name.
5
Results arrive in your IM window within a few seconds.
6
Tap Help in any post-scan dialog to receive this guide link in IM.
📋 Reading the results

Results arrive as RLV Scan: messages in your IM window. The relay status appears first, then the full attachment manifest.

⛔
EXTERNAL OWNER
The attachment is owned by someone other than the resident wearing it. This is the strongest possible signal of external control — someone else's scripted object is on their body.
🔴
RELAY / CTRL
Matches a known RLV relay or controller product by name, description, or script count.
🟡
CHECK
A weaker keyword match. Use the script count in brackets and context to judge significance.
🟢
CLEAN
No flags detected on this item.

The number in brackets after each attachment name is its script count. High counts on flagged items confirm suspicion — OpenCollar v8 typically carries 20–28 scripts.

🚨 Helping the resident escape
1
SAFEWORD — fastest first step Ask the resident to type SAFEWORD in local chat. OpenCollar (all versions), Peanut N°9, and Whim all respond and drop restrictions. OC 8.4 also stops all animations and animlock.
2
Use collar commands if you have access If the resident grants you owner or trusted access, see the Collar Commands section below.
3
Turn off RLVa in Firestorm Guide the resident to: Avatar > Preferences > Firestorm > General — uncheck Allow Remote Scripted Viewer Controls (RLVa) — then relog. Clears all restrictions immediately.
4
Relog on the official SL viewer The official viewer has no RLVa at all. All restrictions clear on login. The resident can then detach flagged items freely.
5
Detach flagged items Right-click the flagged attachment > Detach. If detach is blocked, complete step 3 or 4 first.
6
If control was non-consensual Advise the resident to file an Abuse Report: Help > Report Abuse. Document names, times, and location. Escalate to Senior Mentor if needed.
⌨️ Collar commands

If the resident grants you owner or trusted access to their collar, these chat commands work in local chat. The prefix defaults to the resident's initials — for example if their name is Jane Doe it would be jd.

[prefix] clearremoves ALL restrictions including relay-enforced ones
[prefix] restriction listshows every active restriction
[prefix] rlv offturns the collar relay off (collar restrictions still apply)
[prefix] unsitforces the resident to stand if force-sat

⚠️ Limitations
HUDs worn by others are invisible to SL. They will never appear in the attachment list. The relay ping is the only evidence of a HUD-based relay.
A relay set to Off will not respond to the ping. Results may show clean even if collar-based restrictions are still active on the resident. A relay in Off mode still allows the collar owner to set restrictions directly.
Custom-named relays (e.g. "Pretty Ring") are only caught by the relay ping or by an owner mismatch on the attachment.
Some regions restrict script permissions and may limit attachment enumeration, reducing scan completeness.
